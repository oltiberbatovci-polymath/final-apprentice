# Cloud Infrastructure Deployment with CI/CD Pipeline

## Objective
Design and deploy a complete AWS infrastructure (network, compute, database, and CI/CD) using Terraform.

## Scope
- Create AWS infrastructure (VPC, subnets, routing, IGW, NAT Gateway)
- Deploy a sample web app (Nginx or Flask) on ECS
- Implement CI/CD pipeline using AWS Code Pipeline
- Configure CloudWatch monitoring and alerting
- Document operations via a detailed Runbook

## Execution Plan (5 Days)

### Day	Activities	Expected Deliverables
- Day 1 – Initiation	Define team roles, scope, Develop a simple Project Charter.	Project Charter (v1)
- Day 2 – Design & Planning	Create architecture diagram, Terraform module plan, and CI/CD workflow plan.	Architecture Design, Plan Document
- Day 3 – Development	Implement infrastructure IaC, deploy to QA, and validate environment.	Terraform Code, Screenshots, QA Notes
- Day 4 – Stabilization & Delivery Prep	Code freeze, finalize Demo Script and Runbook.	Demo Script, Runbook
- Day 5 – Delivery & Reporting	Deliver sprint demo, prepare Release Notes and Sprint Report.	Release Notes, Sprint Report

## Deliverables
- Project Charter
- Architecture & Plan Document
- Working Serverless API (deployed)
- Demo Script
- Runbook
- Release Notes
- Sprint Report
- Retrospective Report
